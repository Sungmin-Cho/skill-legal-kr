# CLAUDE.md — Legalize KR 프로젝트

> 이 파일은 AI 협업 도구(Claude)가 프로젝트 컨텍스트를 빠르게 파악하기 위한 안내 문서입니다.

---

## 프로젝트 개요

**Legalize KR**은 대한민국 법령 전체를 Git 저장소로 관리하는 공개 데이터 프로젝트입니다.

- **목적**: 모든 한국 법령을 Markdown 파일로 변환하고, 각 개정을 실제 공포일자를 가진 Git commit으로 추적
- **데이터 출처**: [국가법령정보센터 OpenAPI](https://open.law.go.kr)
- **웹사이트**: [legalize.kr](https://legalize.kr)
- **GitHub**: [legalize-kr/legalize-kr](https://github.com/legalize-kr/legalize-kr)
- **라이선스**: 법령 원문은 공공저작물, 저장소 구조/메타데이터는 MIT

---

## 저장소 구조

```
legal-kr/                         ← 워크스페이스 루트 (선택된 폴더)
└── legalize-kr/                  ← 메인 Git 저장소
    ├── .git/                     ← Git 메타데이터
    ├── README.md                 ← 프로젝트 설명서
    └── kr/                       ← 대한민국 법령 디렉토리
        ├── 민법/
        │   ├── 법률.md
        │   └── 시행령.md
        ├── 근로기준법/
        │   ├── 법률.md
        │   ├── 시행령.md
        │   └── 시행규칙.md
        ├── 가족관계의등록등에관한규칙/
        │   └── 대법원규칙.md
        └── {법령명(띄어쓰기 제거)}/
            └── {문서유형}.md
```

법령 파일은 `kr/` 아래 500개 이상의 디렉토리로 구성됩니다.

## 파일 명명 규칙

| 문서 유형 | 파일명 | 설명 |
|-----------|--------|------|
| 법률 | `법률.md` | 국회에서 제정·개정하는 법률 |
| 시행령 | `시행령.md` | 법률의 시행령 (부모 법률 있음) |
| 시행규칙 | `시행규칙.md` | 법률의 시행규칙 (부령) |
| 대통령령 | `대통령령.md` | 독립 대통령령 (규정, 직제 등 — 부모 법률 없음) |
| 대법원규칙 | `대법원규칙.md` | 대법원에서 제정하는 규칙 |
| 기타 부령 | `{부처명}령.md` | 총리령, 국가보훈부령 등 |

**디렉토리명**: 법령명에서 띄어쓰기를 제거 (예: `친일반민족행위자재산의국가귀속에관한특별법`)

**Unicode 정규화**: `·` (U+00B7) → `ㆍ` (U+318D) 변환 적용 (예: `10·27` → `10ㆍ27`)

## 법령 파일 구조 (YAML Frontmatter)

각 `.md` 파일은 YAML 메타데이터 + Markdown 본문으로 구성됩니다.

```yaml
---
제목: 민법
법령MST: 284415           # 법령 고유 식별자
법령ID: "001001"
법령구분: 법률             # 법률 | 대통령령 | 시행령 | 대법원규칙 등
법령구분코드: A0002
소관부처:
  - 법무부               # 복수 부처 가능 (YAML 리스트)
공포일자: 2024-01-02
공포번호: "19834"
시행일자: 2024-07-03
법령분야: 민사
상태: 시행               # 시행 | 폐지
출처: https://www.law.go.kr/법령/민법
---

# 민법

## 제1편 총칙
### 제1장 통칙

##### 제1조(법원)
민사에 관하여 법률에 규정이 없으면 관습법에 의하고...
```

### 주요 메타데이터 필드

| 필드 | 타입 | 설명 |
|------|------|------|
| `제목` | string | 법령명 (정규화 적용) |
| `원본제목` | string | API 원본 법령명 (정규화 결과와 다를 경우만) |
| `법령MST` | number | 법령 고유 식별자 |
| `법령ID` | string | 법령 분류 코드 |
| `법령구분` | string | 문서 종류 |
| `소관부처` | list | 담당 부처 (복수 가능) |
| `공포일자` | date | 공포일 (YYYY-MM-DD) |
| `공포번호` | string | 공포 번호 |
| `시행일자` | date | 시행일 (YYYY-MM-DD) |
| `법령분야` | string | 법령 분야 (민사, 행정, 노동 등) |
| `상태` | string | 시행 / 폐지 |
| `출처` | url | 국가법령정보센터 원문 URL |

## Git 커밋 구조

```
법률: 민법 (일부개정)

법령 전문: https://www.law.go.kr/법령/민법
제개정문: https://www.law.go.kr/법령/제개정문/민법/(12345,20260317)
신구법비교: https://www.law.go.kr/법령/신구법비교/민법

공포일자: 2026-03-17 | 공포번호: 12345
소관부처: 법무부
법령분야: 민사
법령MST: 284415
```

- 커밋 날짜 = 법령 공포일자 (author/committer date)
- Unix Epoch(1970-01-01) 이전 법령은 커밋 날짜가 1970-01-01로 고정 → 실제 공포일자는 `공포일자` 필드 참조

## 주요 활용 패턴

```bash
# 특정 법령 열람
cat legalize-kr/kr/민법/법률.md

# 법령 개정 이력 확인
git -C legalize-kr log -- kr/민법/

# 전체 법령에서 키워드 검색
grep -r "개인정보" legalize-kr/kr/

# 특정 날짜 기준 법령 상태 확인
git -C legalize-kr log --before="2025-01-01" -1 -- kr/민법/법률.md

# 현재 시행 중인 법령만 필터
grep -rl "상태: 시행" legalize-kr/kr/
```

## 주의사항

- 파이프라인 개선 시 전체 히스토리를 재구성하는 **force-push**가 발생할 수 있음
  → fork/참조 시 `git fetch --all && git reset --hard origin/main` 으로 재동기화 필요
- 법령 본문은 소프트웨어 코드가 아닌 **법령 텍스트 데이터**이므로 코드 수정보다는 내용 조회/분석이 주요 작업
- 법령 조문 참조 시 `출처` 필드의 국가법령정보센터 URL을 통해 원본 확인 권장

## 결과물 저장 규칙

**이 프로젝트에서 사용자가 질문하거나 분석을 요청하면, 결과물을 반드시 파일로 저장해야 합니다.**

### 저장 위치

모든 결과물은 워크스페이스 루트(`legal-kr/`) 아래 `outputs/` 디렉토리에 저장합니다:

```
legal-kr/
└── outputs/
    ├── 민법_분석_20260412.md
    ├── 근로기준법_요약_20260412.docx
    ├── 개인정보보호법_비교표_20260412.xlsx
    └── ...
```

### 파일명 규칙

```
{법령명}_{작업유형}_{날짜YYYYMMDD}.{확장자}
```

예시:
- `민법_조문분석_20260412.md`
- `근로기준법_개정이력_20260412.md`
- `개인정보보호법_요약보고서_20260412.docx`

### 결과물 형식 기준

- **기본 형식은 `.md`** — 모든 결과물은 특별한 요청이 없으면 Markdown 파일로 저장합니다.
- **사용자가 형식을 명시한 경우**에만 아래 형식으로 저장합니다:

| 사용자 요청 예시 | 저장 형식 |
|-----------------|-----------|
| "워드 파일로", "docx로" | `.docx` |
| "엑셀로", "표로 정리", "xlsx로" | `.xlsx` |
| "PPT로", "슬라이드로", "pptx로" | `.pptx` |
| "PDF로" | `.pdf` |

- 저장 후에는 반드시 파일 링크를 제공하여 사용자가 바로 열 수 있게 합니다.
- `outputs/` 디렉토리가 없으면 자동으로 생성합니다.

## 스킬 활용 가이드

> ⚠️ **필수 규칙**: 이 프로젝트에서 사용자가 질문하거나 작업을 요청하면, **반드시 아래 legal 스킬을 먼저 호출한 후 진행**해야 합니다. 스킬을 거치지 않고 바로 답변하는 것은 허용되지 않습니다.

### 작업 유형별 스킬 매핑

| 요청 유형 | 사용할 스킬 |
|-----------|-------------|
| 법령 분석 · 조문 해설 · 법률 질의 | `legal:brief` (topic 모드) |
| 계약서 · 법령 문서 검토 | `legal:contract-review` |
| 규정 준수 · 컴플라이언스 확인 | `legal:compliance-check` |
| 법적 위험 평가 | `legal:legal-risk-assessment` |
| 회의 준비 · 법률 미팅 브리핑 | `legal:meeting-briefing` |
| NDA 검토 · 분류 | `legal:triage-nda` |
| 법률 답변 초안 작성 | `legal:respond` |
| 문서 작성 (Word) | `anthropic-skills:docx` |
| 문서 작성 (PPT) | `anthropic-skills:pptx` |
| 데이터 분석 · SQL | `data:sql-queries`, `data:data-exploration` |
| 번역 | `anthropic-skills:doc-translator` |

### 스킬 호출 원칙

1. **질문·분석 요청** → 먼저 적절한 `legal:*` 스킬을 호출
2. **스킬 실행 후** → legalize-kr 저장소의 법령 파일을 직접 참조하여 조문 근거 보강
3. **결과물 저장** → `outputs/` 디렉토리에 파일로 저장 후 링크 제공

법령 파일 경로는 항상 `legalize-kr/kr/{법령명}/` 형식을 따릅니다.
