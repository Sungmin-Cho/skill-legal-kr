# legal-kr 스킬: 인터뷰 기반 법률 상담 설계

## 요약

legal-kr 스킬의 워크플로우를 변경하여, 상담형 질문에 대해 즉시 답변하지 않고 인터뷰를 통해 충분한 사실관계를 수집한 후 법적 조언을 제공한다.

## 설계 결정 사항

| 항목 | 결정 |
|------|------|
| 적용 범위 | 상담형 질문에만 적용. 단순 법령/조문/판례 조회는 바로 답변 |
| 질문 방식 | 한 번에 하나씩, 변호사 상담 스타일 |
| 분석 전환 기준 | AI가 충분하다고 판단하면 사용자에게 확인 후 전환 |
| 건너뛰기 처리 | 부족한 정보를 한 번 안내한 뒤 사용자 의사 존중 |
| 질문 횟수 상한 | 없음. 충분하다고 판단되면 바로 전환 확인 |

## 요청 분류 기준

### 단순 조회 (인터뷰 없이 바로 진행)

- 특정 법령명/조문 번호를 지정한 요청 ("민법 제750조 알려줘")
- 법령/판례 검색을 명시적으로 요청 ("주택임대차보호법 관련 판례 찾아줘")
- 법령 내용 해설/비교 요청 ("근로기준법 시행령 검색해줘")

### 상담형 (인터뷰 진행)

- 자신의 상황을 설명하며 조언을 구하는 요청 ("교통사고 났는데 배상받을 수 있나요?")
- 법적 판단/대응 방안을 묻는 요청 ("해고당했는데 부당해고인가요?")
- 계약/분쟁 등 구체적 사안에 대한 의견을 구하는 요청 ("집주인이 보증금을 안 돌려줘요")

## 인터뷰 루프 설계

### 진행 방식

1. 사용자의 초기 입력을 분석하여 가장 핵심적인 사실관계부터 질문 (한 번에 하나)
2. 답변을 받으면 지금까지 모인 정보를 내부 정리하고 다음 필요한 질문을 판단
3. 충분성 판단 기준 항목이 갖춰지면 전환 확인

### 충분성 판단 기준

- 법률 분야 (민사, 형사, 가사, 근로, 행정 등)
- 당사자 관계 (누가 누구에게)
- 핵심 사실관계 (무슨 일이 있었는지)
- 시기/금액 등 구체적 수치 (해당되는 경우)
- 사용자가 원하는 결과 (배상, 해지, 대응 방안 등)

### 전환 확인

충분하다고 판단되면:
> "말씀해주신 내용을 정리하면 [요약]입니다. 이 내용을 바탕으로 법령과 판례를 검색하여 분석해드릴까요?"

### 건너뛰기 처리

사용자가 "바로 분석해줘" 등 요청 시:
> "아직 [부족한 항목]을 확인하지 못해 분석의 정확도가 낮을 수 있습니다. 그래도 현재 정보로 진행할까요?"

사용자가 동의하면 바로 분석 전환.

## 워크플로우 변경

### 변경 전 (6단계)

```
Step 1: 데이터 최신화
Step 2: 상황 분석
Step 3: 법령 검색
Step 4: 판례 검색
Step 5: 종합 분석
Step 6: 출력
```

### 변경 후 (7단계)

```
Step 1: 데이터 최신화           (변경 없음)
Step 2: 요청 분류 및 인터뷰     (신규)
  2-1. 단순 조회 / 상담형 분류
  2-2. (상담형) 인터뷰 루프
  2-3. 전환 확인
Step 3: 상황 분석               (기존 Step 2, 중복 질문 문구 제거)
Step 4: 법령 검색               (기존 Step 3)
Step 5: 판례 검색               (기존 Step 4)
Step 6: 종합 분석               (기존 Step 5)
Step 7: 출력                    (기존 Step 6)
```
